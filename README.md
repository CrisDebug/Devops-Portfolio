# DevOps Portfolio – Cristian Barahona

Este repositorio documenta mi aprendizaje práctico en DevOps, mostrando la evolución de una arquitectura desde despliegues clásicos en máquinas virtuales hasta soluciones basadas en contenedores y servicios cloud.

El enfoque es técnico y práctico, con evidencia real de pipelines, infraestructura y automatización implementadas en Azure.

---

## Etapa 1 – CI/CD clásico en VM Linux

Implementación de un pipeline CI/CD utilizando Azure DevOps Pipelines
con un agente self-hosted instalado en una máquina virtual Linux (Ubuntu).

El pipeline automatiza el despliegue de una aplicación web estática
sobre Apache, validando el flujo completo de integración y despliegue
continuo en un entorno clásico basado en VM.

### Tecnologías utilizadas

- Azure DevOps (Pipelines y Environments)
- Agente self-hosted
- Máquina Virtual Linux (Ubuntu)
- Apache HTTP Server
- Git

### Evidencia

La evidencia de esta etapa se encuentra en la carpeta
[`etapa-1-CI-CD-VM/Evidencia`](./etapa-1-CI-CD-VM/Evidencia).
